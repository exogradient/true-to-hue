# Design Log

## Decisions

- **2026-03-31** — OG social preview and tagline. Added `og.png` (1200×630) — 3D parchment card with paint splashes in mode-card colors (pink/blue/mint/gold on dark ground). Full Open Graph + Twitter Card meta tags in `index.html`. Tagline updated from "Color memory game that teaches color perception" to "Unleash your color superpower" across `index.html`, `manifest.json`, and `CLAUDE.md`. Source image (`og.jpeg`) kept alongside for future re-crops.
- **2026-03-31** — Call It mode shipped. XKCD color survey (949 names), lazy LAB cache, CIEDE2000 nearest-match. Key design decisions: (1) Correct name pick = perfect 10, wrong picks scored by CIEDE2000 distance — avoids the "right answer, imperfect score" frustration. (2) Distractor spread: 2 hard (ranks 1-4) + 2 medium (ranks 8-20) + 3 far (ranks 40-99) — pure close distractors made the game too easy since any nearby name scores well. (3) Result cards: no per-card score (irrelevant for naming), stacked layout with correct name + "you said" for wrong picks. (4) Each color name links to its xkcd survey entry via `#:~:text=` fragment. (5) Results eyebrow shows "Call It · name matching" (not "no picker" — internal language). (6) 8 choices in 2×4 grid, 1rem+ font for mobile readability. (7) Dataset in separate `xkcd-colors.js` (same split rationale as `analytics.js` — auditable reference data).
- **2026-03-30** — Picker and button polish pass. Sliders picker redesigned: gradient tracks wrapped in `.slider-track-wrap` containers with inset shadow, rounded corners, and full-height 6px white pill thumbs with glow ring (`box-shadow: 0 0 0 3px rgba(255,255,255,0.25)`). Field picker hue bar expanded to 44px, SB thumb enlarged to 26px — all interactive handles now meet Apple HIG 44px touch target minimum. Glass-morphism overlay button pattern established: 44×44px pills with `backdrop-filter: blur(12px) saturate(1.4)`, dark semi-transparent background, white border, positioned absolute inside their parent card. Applied to confirm button (top-right of guess swatch in `.play-stage-row`) and reveal navigation (home top-left ghost, forward top-right inside `.reveal-card`). Reveal back arrow changed from `icon-back` to `icon-home` — action is quit-to-menu, not browser-back. Results grid locked to `repeat(3, 1fr)` — `auto-fit` was allowing 4 columns on wider phones. Key reject: floating buttons outside cards ("pre-alpha quality"), bottom-right overlay (blocked brightness slider).
- **2026-03-30** — Game screen finalization (pre-launch design pass). Results screen: restored hero panel containment (surface background, border, card shadow, border-radius) and unhid essential metadata — score tier label, "/50" denominator, mode + picker eyebrow, per-card verdict text. Match It pick screen: unhid "TARGET" / "YOUR GUESS" swatch labels so first-time players can tell which is which. Reveal screen: made round progress ("1 / 5") visible, switched from hidden eyebrow to `.pick-progress` class. Pick screen: restored mode eyebrow ("MEMORY", "MATCH"). Layout: top-aligned reveal, results, and history screens (`flex-direction: column; justify-content: flex-start`) — vertical centering was pushing sparse content into awkward mid-screen gaps. Result card grid: bumped minimum width from 120px to 150px for better readability. Feedback text, analysis hints, and color name labels remain hidden — shipped as progressive disclosure in advanced view.
- **2026-03-30** — Alpha scoring gate accepted: ship `effective_delta_guard` as the live scorer. The shipped shape is an effective-ΔE sigmoid `10 / (1 + (ΔEeff/10)^2.25)` with hue recovery `33°/55%`, same-hue SB guard `0.25`, mid-hue guard `0.30`, and same-hue rescue disabled. Shared scorer parity now holds across `public/index.html`, `tools/calibration.jsx`, and `tools/run_calibration.py`, and `make calibrate-release` passed on a fresh post-port dogfood export.
- **2026-03-30** — Auto-grader false-positive correction: narrowed `too_high_same_hue_large_sb` so it only fires for clearly overgenerous same-hue / low-ΔE cases (`score >= 7`, `ΔE <= 12`). Trigger was a dogfood batch where most reviewed rows flipped from `too_high` to `ok`; the narrower rule stopped those overrides from poisoning the release gate.
- **2026-03-30** — Design audit (release-readiness gap analysis): Validated as strong foundations — do not change: achromatic discipline, token architecture, motion timing (150/200/300ms + Rauno/Emil curves), scoring visualization (diagonal clip-path, per-dimension HSB bars, tier system), font choice (Plus Jakarta Sans), SB field + hue bar picker, privacy-first analytics, edge-to-edge memorize, 2-column desktop play layout, panel border/elevation system. Already shipped since audit baseline: reduced-motion support (opacity-only at `prefers-reduced-motion`), Play Again menu bypass. Remaining gaps: 5 release blockers and 10 polish items elevated to `specs-roadmap` Alpha/Beta frontend sections.
- **2026-03-31** — Split It mode shipped. Blind parametric encoding: player sees target color, estimates H/S/B on neutral gray sliders with no color preview or gradient hints. Orientation cues: hue color dots at key positions (0°/30°/60°/120°/180°/240°/300°/330°), opacity-ramping intensity dots for saturation, grayscale dots for brightness. Live value readout tracks thumb position. Reveal shows HSB comparison bars by default (other modes require tap). CIEDE2000 scoring reused from existing pipeline. Key design rejections: gradient-filled tracks (too easy — defeats blind estimation), text labels on hue ticks (unnecessary with color dots), "Muted/Vivid" and "Dark/Bright" endpoint text (dots convey intensity without jargon), separate target label (redundant — the swatch is obvious), field picker option (2D SB plane reveals too much about combined color).
- **2026-03-30** — Home screen redesign: Paint-chip card stack. Modes rendered as overlapping colored swatch cards — Play is the largest card, secondary modes (Match It, Picture It, Call It, Split It) fan out below (mobile) or to the right (desktop) with negative margins and z-index stacking. Same deck, two orientations. Play card color: Parchment (`rgb(238,228,215)` → `rgb(235,225,212)` gradient) — warm canvas that reads as neutral ground, dark text. Secondary card colors locked: Match `rgb(240,100,135)`, Picture `rgb(100,145,250)`, Call `rgb(80,210,180)`, Split `rgb(250,195,70)`. Title treatment: "splash of hue" in Instrument Serif italic lives on the Play card (top-left), not as a separate masthead. Desktop: mode labels use `writing-mode: vertical-rl` + `rotate(180deg)`. Extracted to `public/home.css` + `public/home.js` per inline-until-forced-to-split convention — the home screen is now complex enough to warrant separation. Rejects from iteration: text-only menu (current — too plain), scattered color tiles, floating orbs, spectrum strip, single-hue Play card (too similar to Match It), 2×2 grid desktop layout, horizontal tabs, disconnected vertical stacking.
- **2026-03-30** — Auto-grader framing: human review is primarily meta-eval for the `auto-grader`, not direct scorer labeling. The grader should emit structured verdicts (`too_high`, `too_low`, `ok`, `abstain`) with traceable rule lineage; scorer tuning should consume grader outputs, while random audits remain mandatory to surface blind spots and preserve a path to probabilistic weak-supervision upgrades later.
- **2026-03-30** — Scoring calibration framing: `tools/calibration.jsx` is an assisted dogfooding collector, not a blind annotation workflow. Bias is intentional: pre-label likely failures, overweight hard negatives and boundary cases, and include positives mostly as sanity checks. Goal is fast self-collection during dogfooding, then promotion of selected examples into a locked regression/eval suite that behaves like scorer unit tests.
- **2026-03-30** — PWA icon identity: ship a complete spectral ring with accurate hue order (`0°` red at the top), neutral core, and a small on-ring selector bead that reads as "chosen hue." Key rejections from iteration: broken rings read like loading UI; black pointers read like hardware; encoding brightness in the center makes the mark feel like a picker diagram; eye-like symbolism felt tacky/scary; discrete rounded capsules were less durable than a smoother continuous band. Final split: installed app icon keeps a self-contained plate, with the light plate active and the dark plate preserved as an alternate source; browser favicon drops the plate and simplifies to a more prominent spectral ring. Safari fallback PNGs must be exported with true transparency — white-matted fallbacks break the tab mark.
- **2026-03-30** — Mobile viewport rule: Android-clean is not enough. iPhone Safari needs explicit `viewport-fit=cover`, safe-area-aware bottom padding, and top-aligned flexible gameplay layouts for tall picker screens. Centered vertical layouts can hide the confirm button below the fold even when Pixel renders fine.
- **2026-03-29** — HSB slider visualization for power users. Three gradient bars (H/S/B) with dual pins (target = solid white, guess = translucent) showing where each value landed. Used on both the Reveal screen (tap score to toggle) and Results cards (advanced view). Replaces raw ΔL'/ΔC'/ΔH' delta bars — same information but in the HSB mental model the player actually thinks in.
- **2026-03-29** — Picture It mode: click-to-submit. Selecting a choice immediately confirms — no separate confirm button. Reduces tap count and removes wasted button real estate.
- **2026-03-29** — Picture It target is HSB text, not a swatch. The text IS the game mechanic — Decoding skill means reading "H210 S80 B60" and mentally constructing the color. Showing a swatch would reduce it to a trivial color-matching exercise. See `identity-model` Decoding skill.
- **2026-03-29** — Scoring curve tightened: k=20 → k=10.5 in sigmoid `10 / (1 + (ΔE/k)^2.5)`. Original curve was too generous — ΔE 7.5 scored 9.2 at k=20. Intermediate k=12 still too generous; refined to k=10.5 through calibration dogfooding. Tier thresholds adjusted: 42/35/25/15 (was 47/40/32/22).
- **2026-03-29** — Hue-recovery bonus added. CIEDE2000 weights hue ~1.86x over chroma for saturated colors but treats lightness equally. Research in `pit/2026-03-28-scoring-algorithm` specified hue-weighted scoring. Fix: if hue within 33°, recover up to 50% of lost points. Same-hue-family guesses with brightness/saturation drift get partial credit. Thresholds refined from initial 25°/40% through calibration dogfooding.
- **2026-03-29** — Reveal screen design: side-by-side color panels (not diagonal split), score colored to match target, verdict text only — no ΔE or HSB numbers. Rationale: ΔE is jargon, HSB values are redundant when you can see the colors. The visual comparison + score + verdict communicate everything.
- **2026-03-29** — Results screen: simplified decorative treatment. ~~Bare numbers directly on background~~ evolved to lightweight panel/raised-panel surfaces with subtle borders. Raw HSB text dump removed from advanced detail view — replaced with HSB slider visualization (per 2026-03-29 slider decision).
- **2026-03-29** — ~~Vercel SPA rewrite doesn't work with framework detection enabled.~~ Superseded: disabled framework detection (`"framework": null`), added explicit `/` → `/index.html` rewrite. Root now served directly from CDN — no function invocation, no 307 redirect hop. The `@app.get("/")` handler was removed; local dev uses `StaticFiles(html=True)` instead.
- **2026-03-29** — History: localStorage, not server. Sorted by highest score, 20 per mode, tabs per mode. Full game data stored (targets, guesses, scores) so tapping a row shows the complete result view. `/api/history` removed — returns in Phase 2 as a leaderboard endpoint.
- **2026-03-29** — Identity: no accounts, no auth. Privacy-first. Users provide a display name (pseudonym) only when needed (multiplayer/leaderboards), freely changeable each time. Stored in localStorage to prepopulate, not enforced. No server-side user identity until Phase 3 challenges require linking attempts.
- **2026-03-29** — Per-color reveal: after each pick, show target vs guess side-by-side with score. Client computes CIEDE2000 locally for instant feedback (no round-trip latency). Server accepts pre-scored results on final submit — no server-side re-scoring. Flow per round: memorize → pick → reveal. Final results screen still shows all 5 + total.
- **2026-03-29** — Backend architecture: client-driven game flow. Client generates colors and scores locally — no server round-trip to start a game. `submit` accepts targets + guesses + client-computed scores, then appends completed game to DB. Server is append-only persistence for analytics. Rationale: serverless (Vercel) means ephemeral `/tmp`, no shared state across instances, cold starts add latency. Client-side generation eliminates network dependency for gameplay — games start instantly. DB is never in the critical path. Tradeoff: scoring is technically cheatable (client sends scores), but this is a training tool — cheating only hurts yourself.
- **2026-03-29** — Database strategy: Turso (LibSQL) for persistent storage, with `/tmp` SQLite as a zero-dependency fallback. Turso is SQLite-compatible (minimal migration from current schema), edge-distributed, free tier. The append-only write pattern (one INSERT per completed game) is ideal for either backend.
- **2026-03-29** — Refresh resets. No session persistence for in-progress games. A browser refresh returns to the menu — the partially-played round is silently discarded. This is intentional: rounds are short (< 2min), there's no penalty for abandoning, and persisting partial state adds complexity with no training value.
- **2026-03-29** — Schema evolution path for multiplayer: solo games (now) → leaderboards (add `display_name` column on `games`, no users table) → shared challenges (add `challenges` table storing target colors server-side, `challenge_id` FK on `games`). Shared challenges restore server-side truth for competitive scoring while solo play stays stateless.
- **2026-03-28** — Results: clean by default (swatches + scores). HSB values, ΔE, feedback text behind hidden activation (URL param / shortcut). Mirrors dialed.gg's minimal results, but insights available for power users.
- **2026-03-28** — Frontend: evolve own visual identity, not a dialed.gg clone. Use dialed.gg as quality gate — if it doesn't hold up side-by-side, it's not ready.
- **2026-03-28** — ~~Feedback messages: server-side message bank.~~ Superseded: feedback messages are client-side (`colorFeedbackText`, `totalFeedbackText` in index.html). Original plan was server-side for shareability; pivoted to client-side with the stateless architecture.
