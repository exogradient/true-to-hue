## Decisions

### Doc system & architecture
- **Four-layer doc system**: identity / design / specs / meta. Layer encoded as filename prefix.
- **PIT docs**: Frozen research snapshots in `docs/pit/`, not managed.
- **Specs pipeline**: roadmap → features → journey.
- **CIEDE2000 scoring**: Sigmoid `10 / (1 + (ΔE00/20)^2.5)`, per-dimension deltas (ΔL', ΔC', ΔH').
- **Results: clean by default**: Details behind hidden activation (URL param / shortcut).
- **Feedback messages server-side**: Returned with submit response. Enables sharing/leaderboards.

### Visual identity
- **Achromatic constraint**: Current navy/coral violates. Move to near-black `#0a0a0a` with neutral gray ramp.
- **Type is the brand**: Single typeface, single weight (500). Warm geometric or humanist — not Swiss brutalism. Candidates: Satoshi, General Sans, Space Grotesk, Instrument Sans.
- **Evolve own identity, not clone dialed.gg**: Use dialed as quality gate, but different direction: teaching not intimidating.
- **Influence profiles are decision-making tools**: "This profile is needed to fill any ambiguity gaps. They will be making key new decisions." Profiles contain enough to resolve design questions without further research.
- **No implementation without identity**: "You went straight to implementation. This is a key piece of this build and we will need to iterate profusely." Identity doc must be iterated and locked first.
- **SB field picker kept**: Spatial advantage over dialed's three strips. Polish, don't replace.
- **Desktop goes full-width**: Not card-in-viewport. Educational modes need breathing room.

### Skill model
- **Picker fluency is UX, not a skill**: Good UI makes it invisible. Skills are cognitive.
- **Recall → Retention**: Maintenance during delay is the bottleneck, not retrieval.
- **Context added as fifth core skill**: Simultaneous contrast awareness.
- **Dial It In → Picture It**: Names the cognitive operation, fits the verb-it pattern.
- **Judge It mode**: Context skill. Target on neutral background, 4 choices on colored surround. Reveal strips backgrounds. Difficulty: strong surrounds → subtle surrounds → no neutral reference.
- **Scientific basis column**: Precise terminology (chromatic discrimination, color appearance scaling, color visualization, delayed reproduction, simultaneous contrast awareness).

### Voice
- **Honest/direct/fun, not brutal**: Brutal is dialed's voice and doesn't match warm visual identity. Not generic either.
- **Varies by mode, not user setting**: Play gets personality ("Right family, wrong sibling"), skill modes get coaching (per-dimension feedback). Mode selection is the toggle.

## Insights
- **Encoding is genuinely hard**: Untrained observers perceive color holistically. Dimensional decomposition is a learned skill.
- **Transfer is hue-specific**: Training in one color region doesn't transfer. Game should cover color space systematically.
- **Categorical drift during delay**: Color memories pull toward prototypes. HSB encoding counteracts this — the pedagogical core.
- **Sapir-Whorf on color is real**: Learning color vocabulary shifts discrimination at category boundaries. Validates Name It mode.
- **Achromatic UI is consensus**: Every polished color game converges on this.
- **Picker UX is unsolved**: No competitor has a great one. SB field is competitive.
